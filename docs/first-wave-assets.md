# MCGR First-Wave Assets

This file turns the hub recommendations into a practical asset build plan for the first core framework repos.

## Build Targets

### 1. `multi-cloud-governance-model`

Primary asset:

- governance control map

Supporting assets:

- policy layer diagram
- operating model overview
- repository cross-link block

### 2. `slo-driven-cloud-architecture`

Primary asset:

- SLO scorecard

Supporting assets:

- service tier diagram
- reliability target template
- incident threshold checklist

### 3. `cloud-finops-governance`

Primary asset:

- FinOps review template

Supporting assets:

- budget variance checklist
- spend optimization flow
- monthly executive summary format

### 4. `dr-governance-framework`

Primary asset:

- recovery validation checklist

Supporting assets:

- RTO/RPO matrix
- failover test template
- disaster recovery evidence log

### 5. `ai-driven-observability-framework`

Primary asset:

- observability signal map

Supporting assets:

- telemetry taxonomy
- alert-to-action workflow
- anomaly response summary

### 6. `cloud-risk-compliance-controls`

Primary asset:

- control-to-evidence map

Supporting assets:

- audit readiness checklist
- compliance register
- evidence catalog index

## Shared Asset Rules

- Keep the naming consistent across repos.
- Put the primary asset near the top of the repo README.
- Link the primary asset back to `MCGR-Framework`.
- Include one short example per repo.

## Execution Order

1. Create the primary asset for each repo.
2. Add the README template structure.
3. Add the repo cross-link block.
4. Add one example or evidence artifact.
5. Expand supporting assets once the first pass is complete.
